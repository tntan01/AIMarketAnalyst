PROMPT 3 — PHASE 14.3: THÊM HELPER ĐỌC FINAL SCORE, GATE, ENTRY STATUS VÀ DECISION CAP AN TOÀN

Bạn đang làm trong project ai-market-analyst-v2.0 trên VSCode.

Bối cảnh:
Decision engine sẽ nhận dữ liệu từ analysis_result hoặc dict payload. Dữ liệu có thể thiếu, bẩn hoặc dùng tên cũ. Cần helper an toàn trước khi viết logic quyết định.

Mục tiêu riêng của prompt này:
Thêm helper đọc dữ liệu an toàn trong decision_engine. Chưa viết make_final_decision đầy đủ.

Yêu cầu rất quan trọng:
- Chỉ được sửa:
  1. core/decision_engine.py
  2. tests/test_decision_engine_helpers_phase14.py
- Không sửa analysis_engine.py.
- Không sửa final_score_engine.py.
- Không sửa trade_gate_engine.py.
- Không sửa UI/scanner.

Việc cần làm:
1. Trong core/decision_engine.py, thêm:
   def clamp_score(value: object, default: int = 0) -> int:
       ...

   Yêu cầu:
   - Nhận int, float, string số.
   - None, "", "abc", NaN, inf thì trả default.
   - Kết quả trong 0-100.
   - Không raise exception.

2. Thêm:
   def normalize_decision_cap(value: object) -> str | None:
       ...

   Yêu cầu:
   - Nhận None, "", "none" => None.
   - Nhận các value:
     + "TRADE_BLOCKED"
     + "WATCH_ONLY"
     + "WAITING_CONFIRMATION"
     + "READY_TO_TRADE"
     + "STAND_ASIDE"
   - Không phân biệt hoa thường.
   - Chấp nhận alias cũ nếu có:
     + "blocked" => TRADE_BLOCKED
     + "watch" => WATCH_ONLY
     + "wait" => WAITING_CONFIRMATION
     + "ready" => READY_TO_TRADE
     + "stand_aside" => STAND_ASIDE
   - Unknown => None.

3. Thêm:
   def normalize_entry_status(value: object) -> str:
       ...

   Yêu cầu:
   - confirmed_entry giữ nguyên.
   - waiting_confirmation giữ nguyên.
   - watch_zone giữ nguyên.
   - invalidated giữ nguyên.
   - no_setup giữ nguyên.
   - None hoặc unknown => "unknown".
   - Không raise exception.

4. Thêm:
   def gate_allows_trade(gate_result: dict | None, trade_permission: dict | None = None) -> bool:
       ...

   Logic:
   - Nếu gate_result là dict và có key allowed thì bool(allowed).
   - Nếu gate_result không có allowed nhưng trade_permission.status == "blocked" thì False.
   - Nếu không có dữ liệu gate rõ ràng thì False hoặc True?
   - Chọn False để an toàn, nhưng phải warning ở make_final_decision sau.
   - Không raise exception.

5. Thêm:
   def merge_unique_codes(*groups) -> list[str]:
       ...

   Yêu cầu:
   - Nhận list/tuple/set/None.
   - Bỏ None, string rỗng.
   - Deduplicate giữ thứ tự.
   - Không raise exception.

6. Tạo test:

PROMPT 4 — PHASE 14.4: TRIỂN KHAI make_final_decision() THEO GATE, ENTRY STATUS VÀ FINAL SCORE

Bạn đang làm trong project ai-market-analyst-v2.0 trên VSCode.

Bối cảnh:
Đã có decision_engine.py với helper an toàn. Bây giờ triển khai hàm quyết định chính.

Mục tiêu riêng của prompt này:
Tạo make_final_decision() độc lập, không tích hợp vào analysis_engine trong prompt này.

Yêu cầu rất quan trọng:
- Chỉ được sửa:
  1. core/decision_engine.py
  2. tests/test_decision_engine_basic_phase14.py
- Không sửa analysis_engine.py.
- Không sửa signal_engine.py.
- Không sửa final_score_engine.py.
- Không sửa trade_gate_engine.py.
- Không sửa UI/scanner.
- Không đổi decision hiện tại trong app.

Việc cần làm:
1. Trong core/decision_engine.py, thêm hoặc hoàn thiện:
   def make_final_decision(
       *,
       final_score: object,
       gate_result: dict | None = None,
       entry_status: object = None,
       score_gap: object = None,
       trade_permission: dict | None = None,
       thresholds: dict | None = None,
   ) -> dict:
       ...

2. Logic quyết định bắt buộc, theo đúng thứ tự ưu tiên:
   A. Nếu gate_result.allowed là False hoặc trade_permission.status == "blocked":
      decision = TRADE_BLOCKED
      block_codes thêm DECISION_GATE_BLOCKED
      allowed = False
      Không xét final_score nữa.

   B. Nếu gate_result.decision_cap là TRADE_BLOCKED:
      decision = TRADE_BLOCKED
      block_codes thêm DECISION_GATE_BLOCKED
      allowed = False.

   C. Nếu decision_cap là WATCH_ONLY:
      decision = WATCH_ONLY
      warning_codes thêm DECISION_GATE_CAPPED.

   D. Nếu decision_cap là WAITING_CONFIRMATION:
      decision = WAITING_CONFIRMATION
      warning_codes thêm DECISION_GATE_CAPPED.

   E. Nếu score_gap có giá trị và score_gap < min_score_gap:
      decision = WAITING_CONFIRMATION
      warning_codes thêm DECISION_SCORE_GAP_LOW.
      Lý do: hai phe Buy/Sell còn sát điểm, không được READY.

   F. Nếu entry_status != "confirmed_entry":
      - Nếu entry_status là "watch_zone" thì decision = WATCH_ONLY.
      - Nếu entry_status là "waiting_confirmation" thì decision = WAITING_CONFIRMATION.
      - Nếu entry_status là "invalidated" hoặc "no_setup" thì decision = STAND_ASIDE.
      - Nếu unknown thì decision = WAITING_CONFIRMATION.
      warning_codes thêm DECISION_ENTRY_NOT_CONFIRMED nếu chưa confirmed.

   G. Nếu entry_status == "confirmed_entry":
      - final_score >= ready threshold mặc định 80 => READY_TO_TRADE.
      - final_score >= watch threshold mặc định 65 => WATCH_ONLY.
      - final_score >= wait threshold mặc định 50 => WAITING_CONFIRMATION.
      - còn lại => STAND_ASIDE.

3. Output bắt buộc:
   {
       "decision": str,
       "final_score": int,
       "decision_label": str,
       "reason_codes": list[str],
       "warning_codes": list[str],
       "block_codes": list[str],
       "decision_cap": str | None,
       "allowed": bool,
       "score_breakdown": {
           "entry_status": str,
           "score_gap": int | None,
           "thresholds": dict,
           "gate_allowed": bool,
           "applied_rule": str,
       },
       "reason": str,
   }

4. decision_label tiếng Việt:
   - READY_TO_TRADE: "Sẵn sàng giao dịch"
   - WAITING_CONFIRMATION: "Chờ xác nhận"
   - AGGRESSIVE_SETUP: "Setup mạo hiểm"
   - WATCH_ONLY: "Chỉ theo dõi"
   - TRADE_BLOCKED: "Bị chặn giao dịch"
   - STAND_ASIDE: "Đứng ngoài"

5. reason_codes:
   - READY_TO_TRADE thêm DECISION_READY_TO_TRADE và DECISION_FINAL_SCORE_STRONG.
   - WATCH_ONLY thêm DECISION_WATCH_ONLY và có thể DECISION_FINAL_SCORE_MODERATE nếu do score 65-79.
   - WAITING_CONFIRMATION thêm DECISION_WAITING_CONFIRMATION.
   - STAND_ASIDE thêm DECISION_STAND_ASIDE và DECISION_FINAL_SCORE_WEAK nếu do score thấp.
   - TRADE_BLOCKED thêm DECISION_TRADE_BLOCKED.

6. Dữ liệu bẩn không được crash.
7. Không gọi final_score_engine.
8. Không gọi trade_gate_engine.
9. Không gọi service/database/UI.

PROMPT 5 — PHASE 14.5: THÊM HỖ TRỢ AGGRESSIVE_SETUP NHƯ TRẠNG THÁI TÙY CHỌN, KHÔNG MẶC ĐỊNH READY

Bạn đang làm trong project ai-market-analyst-v2.0 trên VSCode.

Bối cảnh:
Plan Phase 14 có state AGGRESSIVE_SETUP. State này chỉ nên xuất hiện khi caller cho phép rõ ràng, ví dụ M15 loose nhưng score rất cao. Không được biến setup mạo hiểm thành READY.

Mục tiêu riêng của prompt này:
Thêm tùy chọn allow_aggressive_setup vào make_final_decision.

Yêu cầu rất quan trọng:
- Chỉ được sửa:
  1. core/decision_engine.py
  2. tests/test_decision_engine_aggressive_phase14.py
- Không sửa entry_engine.py.
- Không sửa analysis_engine.py.
- Không sửa UI/scanner.
- Không đổi gate.

Việc cần làm:
1. Cập nhật chữ ký:
   def make_final_decision(..., allow_aggressive_setup: bool = False, ...):

2. Logic:
   - Nếu allow_aggressive_setup=False thì behavior giữ nguyên như prompt 4.
   - Nếu allow_aggressive_setup=True:
     + entry_status == "waiting_confirmation"
     + final_score >= ready threshold
     + gate allowed
     + decision_cap không phải WATCH_ONLY/WAITING_CONFIRMATION/TRADE_BLOCKED
     + score_gap rõ ràng
     => decision = AGGRESSIVE_SETUP.
   - AGGRESSIVE_SETUP không bao giờ được dùng khi:
     + gate blocked.
     + decision_cap tồn tại.
     + score_gap thấp.
     + entry_status invalidated/no_setup/watch_zone.
   - reason_codes thêm DECISION_AGGRESSIVE_SETUP.
   - warning_codes thêm DECISION_ENTRY_NOT_CONFIRMED.
   - reason phải nói rõ đây là setup mạo hiểm, chưa phải READY.

3. Không thay đổi default allow_aggressive_setup=False.

4. Test:
   tests/test_decision_engine_aggressive_phase14.py

Test bắt buộc:

   - entry_status chưa confirmed không được READY.
   - final_score >= 80 + gate allowed + entry confirmed + gap rõ => READY_TO_TRADE.
   - Dữ liệu bẩn không crash.

5. analysis_engine nếu đã tích hợp:
   - result có decision_engine.
   - Nếu use_decision_engine_action=False thì legacy action không bị thay đổi.
   - Nếu use_decision_engine_action=True thì decision_summary["action"] dùng legacy_action từ decision_engine.

6. Không có thay đổi có chủ đích ở:
   - ui/
   - data/migrations/
   - core/scanner.py
   - core/signal_engine.py
   - core/trade_gate_engine.py
