PROMPT 3: Sửa icon để hiển thị thật, sinh động, không chỉ là ký hiệu ASCII

Bạn là Deepseek_v4_pro đang lập trình trong VSCode tại repo ai-market-analyst-v2.0.

Hiện tại cột icon trong dialog chỉ hiển thị ký hiệu ASCII như #, FX, >>, <>, (+), (S), * nên người dùng nói “Các Icon chưa hiển thị”.

Hãy sửa để icon hiển thị trực quan hơn.

Yêu cầu bắt buộc:
1. Không chỉ dùng ký hiệu ASCII đơn giản.
2. Ưu tiên dùng icon ổn định trong PyQt:
   - Có thể dùng QStyle.StandardPixmap thông qua QApplication.style().standardIcon(...)
   - Có thể dùng QIcon/QPixmap vẽ đơn giản bằng code nếu cần.
   - Có thể dùng emoji Unicode nếu set font phù hợp, nhưng phải có fallback.
3. Cách khuyến nghị:
   - Giữ cột icon trong QTableWidget.
   - Mỗi item icon có thể setIcon(QIcon) và setText("") hoặc setText(icon_text fallback).
   - Đặt table.setIconSize(QSize(22, 22)) hoặc tương đương.
   - Cột icon width khoảng 48-56px.
4. Nếu dùng emoji:
   - Set font cho icon_item là “Segoe UI Emoji” trên Windows nếu có thể.
   - Icon text nên là emoji thật, ví dụ:
     - 🏁, 💱, ▶️, 🧭, 🎯, 🛡️, ⭐, ✅, 🔎, 🗂️, 🚦, ⏱️, ↔️, ▲, ▼, 🌐, 🧲, ⚖️, 💬, 🔍
   - Nếu emoji không render tốt, fallback vẫn phải rõ hơn ASCII, ví dụ icon Qt chuẩn hoặc ký hiệu ▲/▼/✓.
5. Nếu dùng QStyle icons:
   - Tạo mapping helper, ví dụ:
     def _help_icon(self, icon_key: str) -> QIcon:
         style = self.style() or QApplication.style()
         ...
   - Dùng StandardPixmap phù hợp nhất có sẵn.
   - Không cần hoàn hảo 1-1, nhưng phải là icon thật.
6. Có thể để COLUMN_HELP có key:
   - "icon": "🏁" hoặc
   - "icon_key": "rank"
   - Nếu thêm icon_key thì code dễ map QIcon hơn.

Yêu cầu test thủ công:
- Khi render preview, cột đầu tiên không còn chỉ là #, FX, >>, <>.
- Mỗi row có icon hoặc emoji nhìn rõ.
- Không có ô icon trống.
- Không crash nếu hệ thống không có font emoji.

Ràng buộc:
- Không thêm dependency icon ngoài.
- Không tải icon từ internet.
- Không thêm file ảnh nếu không cần.
- Không dùng SVG/asset phức tạp.
- Không sửa theme global.
- Không sửa table chính.
- Không sửa core.
