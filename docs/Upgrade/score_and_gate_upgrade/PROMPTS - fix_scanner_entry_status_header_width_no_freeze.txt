PROMPT 2: Sửa width/min-width riêng cho cột “Trạng thái entry” để header không bị lẹm

Bạn là Deepseek_v4_pro đang lập trình trong VSCode tại repo ai-market-analyst-v2.0.

Dựa trên khảo sát ở prompt trước, hãy sửa phần resize cột của bảng scanner.

Mục tiêu:
- Header “Trạng thái entry” phải hiển thị đầy đủ, không bị cắt chữ đầu/cuối.
- Nội dung trong cột vẫn hiển thị tiếng Việt ngắn như:
  - Sẵn sàng
  - Chờ xác nhận
  - Theo dõi vùng
  - Vô hiệu
  - Thiếu dữ liệu
- Không làm bảng mất scroll ngang.
- Không làm cột này rộng quá mức.

Yêu cầu kỹ thuật:
1. Không hard-code index cột entry_status bằng số nếu có thể tránh.
   - Nên tạo helper hoặc dict dựa trên key:
     column_keys = [key for key, _ in self.table_model.COLUMNS]
     entry_status_col = column_keys.index("entry_status")
   - Nếu cần dùng dict cấu hình theo key thì càng tốt.
2. Ưu tiên tận dụng cơ chế hiện có:
   - TABLE_EXTRA_COLUMN_PADDING
   - _content_width_for_column()
   - _configure_table_columns()
3. Có thể thêm min width riêng cho cột entry_status, ví dụ:
   - 150px hoặc 160px, tùy font thực tế.
   - Width cuối cùng nên là max(content_width, min_width).
4. Nếu hiện code dùng TABLE_EXTRA_COLUMN_PADDING theo index, cần tránh index dễ sai khi thứ tự cột thay đổi.
   - Có thể thêm một cấu hình mới theo key, ví dụ:
     TABLE_MIN_COLUMN_WIDTHS_BY_KEY = {"entry_status": 150}
     TABLE_EXTRA_COLUMN_PADDING_BY_KEY = {"entry_status": 32}
   - Hoặc nếu muốn sửa ít nhất, vẫn có thể thêm index đúng vào TABLE_EXTRA_COLUMN_PADDING, nhưng phải đảm bảo test phát hiện nếu index sai.
5. _content_width_for_column() cần tính được header text + padding đủ rộng.
6. Nếu sửa _content_width_for_column(), không làm ảnh hưởng cột “Lý do chính” và các cột khác.

Ràng buộc cực quan trọng:
- Không triển khai freeze column.
- Không thêm table view phụ.
- Không override paint event phức tạp.
- Không đổi header “Trạng thái entry” thành “TT entry”, vì người dùng đã đồng ý giữ tên rõ nghĩa.
- Không đổi key dữ liệu "entry_status".
- Không đổi display mapping tiếng Việt của entry_status nếu đã có.
- Không đổi thứ tự cột.
- Không sửa core.

Kiểm tra thủ công bằng Python sau khi sửa:
- Tạo ScannerScreen offscreen.
- Tìm index cột entry_status.
- Gọi _configure_table_columns().
- Lấy width thực tế:
  width = screen.table.horizontalHeader().sectionSize(entry_status_col)
- Lấy header text width:
  text_width = screen.table.horizontalHeader().fontMetrics().horizontalAdvance("Trạng thái entry")
- Assert:
  width >= text_width + một khoảng padding an toàn, ví dụ 24-32px.
- Assert header vẫn là “Trạng thái entry”.

Sau khi sửa xong, tự rà lại:

- Cột entry_status có quá rộng không?
