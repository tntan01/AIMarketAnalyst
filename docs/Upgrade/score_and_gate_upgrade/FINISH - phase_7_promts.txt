```python
def calculate_loss_stats(
    closed_trades: list[dict],
    now=None,
    timezone_name: str = "Asia/Ho_Chi_Minh",
) -> dict:
    ...
```

Input `closed_trades` là list dict, mỗi trade có thể có:
- `result_pct`
- `result_r`
- `closed_at`
- `exit_reason`
- `symbol`
- `direction`
- `actual_lot`
- `planned_lot`

Output bắt buộc:

```python
{
    "daily_result_pct": 0.0,
    "weekly_result_pct": 0.0,
    "consecutive_losses": 0,
    "daily_trade_count": 0,
    "weekly_trade_count": 0,
}
```

Logic:
- Chỉ tính trade đã đóng, có `closed_at`.
- Nếu thiếu `result_pct` thì coi là 0.0.
- `daily_result_pct`: tổng `result_pct` các trade trong ngày.
- `weekly_result_pct`: tổng `result_pct` các trade trong tuần.
- `consecutive_losses`: đếm chuỗi lệnh thua liên tiếp gần nhất, dựa trên `result_pct < 0` hoặc nếu thiếu thì `result_r < 0`.
- Nếu input thiếu key/format lỗi thì không crash.

Bước 6 — Check account guard:
Thêm hàm chính:

```python
def check_account_guard(
    closed_trades: list[dict] | None = None,
    open_trades: list[dict] | None = None,
    settings: dict | None = None,
    action: str = "open_new_trade",
    now=None,
) -> dict:
    ...
```

Default settings:

```python
max_daily_loss_pct = 2.0
max_weekly_loss_pct = 5.0
max_consecutive_losses = 3
max_open_risk_pct = 3.0
trader_timezone = "Asia/Ho_Chi_Minh"
```

Output bắt buộc:

```python
{
    "allowed": True,
    "blocked": False,
    "action": "open_new_trade",
    "block_codes": [],
    "warning_codes": [],
    "reasons": [],
    "stats": {
        "daily_result_pct": ...,
        "weekly_result_pct": ...,
        "consecutive_losses": ...,
        "open_risk_pct": ...,
    },
}
```

Logic:
- Nếu action thuộc `RISK_REDUCING_ACTIONS` thì luôn `allowed=True`, nhưng vẫn trả warning/stats nếu có vi phạm.
- Nếu action thuộc `RISK_INCREASING_ACTIONS` thì check:
  1. daily_result_pct <= -max_daily_loss_pct:
     - block code `"DAILY_LOSS_LIMIT_REACHED"`
  2. weekly_result_pct <= -max_weekly_loss_pct:
     - block code `"WEEKLY_LOSS_LIMIT_REACHED"`
  3. consecutive_losses >= max_consecutive_losses:
     - block code `"MAX_CONSECUTIVE_LOSSES_REACHED"`
  4. open_risk_pct >= max_open_risk_pct:
     - block code `"MAX_OPEN_RISK_REACHED"`
- Nếu có block code và action tăng rủi ro:
  - `allowed=False`
  - `blocked=True`
- Reasons tiếng Việt ngắn gọn, ví dụ:
  - "Đã chạm giới hạn lỗ trong ngày, không được mở thêm lệnh mới."
  - "Tổng rủi ro lệnh đang mở đã vượt giới hạn cho phép."

Yêu cầu an toàn:
- Không crash nếu `closed_trades=None`, `open_trades=None`, `settings=None`.
- Không crash nếu `closed_at` là string ISO hoặc datetime.
- Nếu parse date lỗi thì bỏ qua trade đó khỏi daily/weekly count.
- Không dùng pandas.
- Không dùng dependency ngoài.
