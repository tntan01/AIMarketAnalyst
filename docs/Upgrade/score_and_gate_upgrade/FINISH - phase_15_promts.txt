PROMPT 5 — PHASE 15.5: TRIỂN KHAI calculate_opportunity_score() ĐỘC LẬP

Bạn đang làm trong project ai-market-analyst-v2.0 trên VSCode.

Bối cảnh:
Phase 15 cần opportunity_score để ranking scanner không chỉ dựa vào best_score. Công thức MVP:
opportunity_score = final_score + proximity_bonus + readiness_bonus + rr_bonus - spread_penalty - news_penalty

Mục tiêu riêng của prompt này:
Tạo calculate_opportunity_score() độc lập trong scanner_ranking_engine. Chưa tích hợp vào scanner.py.

Yêu cầu rất quan trọng:
- Chỉ được sửa:
  1. core/scanner_ranking_engine.py
  2. tests/test_scanner_ranking_engine_opportunity_phase15.py
- Không sửa core/scanner.py.
- Không sửa decision_engine.py.
- Không sửa final_score_engine.py.
- Không sửa UI/controller.

Việc cần làm:
1. Trong core/scanner_ranking_engine.py, thêm:
   def calculate_opportunity_score(row: dict | None, *, weights: dict | None = None) -> dict:
       ...

2. Input row có thể chứa:
   - final_score
   - best_score
   - scanner_action
   - scanner_decision hoặc decision
   - trade_permission
   - entry_status
   - price_vs_zone
   - risk_reward
   - expected_effective_rr
   - spread_status
   - news_in_3h
   - high_impact_event_within_30m
   - ready_to_trade

3. Logic tính score:
   - base_final_score:
     + ưu tiên row["final_score"]
     + fallback row["best_score"]
     + fallback 0.

   - scanner_group = classify_scanner_group(...).

   - proximity_bonus:
     + price_vs_zone == in_zone: +8
     + near_zone: +4
     + far/unknown: +0

   - readiness_bonus:
     + scanner_group == READY_NOW: +10
     + scanner_group == WAITING_CONFIRMATION: +3
     + scanner_group == WATCH_ZONE: +0
     + scanner_group == BLOCKED: +0

   - rr_bonus:
     + rr >= 2.0: +5
     + rr >= 1.5: +3
     + rr >= 1.3: +1
     + rr < 1.3: +0 và penalty code SCANNER_RR_WEAK.
     Dùng expected_effective_rr nếu có và > 0, ngược lại parse risk_reward.

   - spread_penalty:
     + spread_status == abnormal: -8 và penalty SCANNER_SPREAD_PENALTY.
     + spread_status == caution: -4.
     + normal/unknown: 0.

   - news_penalty:
     + high_impact_event_within_30m True: -10 và penalty SCANNER_NEWS_PENALTY.
     + news_in_3h True: -5.
     + không có tin: 0.

   - Nếu scanner_group == BLOCKED:
     + opportunity_score không được cao hơn 20, dùng min(score, 20).
     + warning hoặc penalty phù hợp.

   - Clamp opportunity_score trong 0-120. Lý do cho phép >100 nhẹ vì bonus proximity/readiness.

4. Output bắt buộc:
   {
       "opportunity_score": int,
       "scanner_group": str,
       "reason_codes": list[str],
       "warning_codes": list[str],
       "penalty_codes": list[str],
       "score_breakdown": {
           "base_final_score": int,
           "proximity_bonus": int,
           "readiness_bonus": int,
           "rr_bonus": int,
           "spread_penalty": int,
           "news_penalty": int,
           "raw_score": float,
       },
       "reason": str,
   }

5. Reason codes:
   - Luôn thêm SCANNER_OPPORTUNITY_SCORE_OK nếu tính được.
   - Group ready_now thêm SCANNER_RANKING_READY_NOW.
   - Group waiting_confirmation thêm SCANNER_RANKING_WAITING_CONFIRMATION.
   - Group watch_zone thêm SCANNER_RANKING_WATCH_ZONE.
   - Group blocked thêm SCANNER_RANKING_BLOCKED.
   - price in_zone thêm SCANNER_PROXIMITY_IN_ZONE.
   - near_zone thêm SCANNER_PROXIMITY_NEAR_ZONE.
   - far thêm SCANNER_PROXIMITY_FAR.
   - rr >= 2 thêm SCANNER_RR_STRONG.

6. Dữ liệu bẩn không crash.
7. Không mutate row input.

Test:
