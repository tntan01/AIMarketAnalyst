FINISH - PROMPT 2 — SỬA BACKEND: THÊM signal_score, GIỮ total
=========================================================

Bạn là senior Python engineer. Hãy thực hiện Phase 1 theo phạm vi rất hẹp.

Mục tiêu:
Thêm `signal_score` vào output scoring, nhưng vẫn giữ `total` để tương thích ngược.

Phạm vi được phép sửa:
- `core/signal_engine.py`
- `core/analysis_engine.py` nếu đang đọc trực tiếp `total`
- Test backend liên quan nếu có

Không được sửa:
- Không sửa UI ở prompt này.
- Không đổi công thức scoring.
- Không đổi trọng số scoring.
- Không đổi threshold decision.
- Không đổi tên decision/status.
- Không refactor code không liên quan.
- Không format lại toàn file.
- Không xóa key `total`.

Yêu cầu chi tiết:
1. Trong `score_scenario()` hoặc function tương đương đang trả về `"total": total`, thêm:
   ```python
   "signal_score": total,
   "total": total,  # deprecated, kept for backward compatibility
   ```
2. Nếu có nơi đọc score để xử lý tiếp, đổi cách đọc sang:
   ```python
   score.get("signal_score", score.get("total", 0))
   ```
3. Chỉ đổi những nơi liên quan trực tiếp tới việc đọc điểm scoring.
4. Giữ nguyên hành vi decision hiện tại.
5. Không thay đổi output cũ ngoài việc bổ sung thêm `signal_score`.
6. Nếu có test hiện tại đang assert `total`, không xóa test đó; chỉ bổ sung test kiểm tra `signal_score`.

Tiêu chí hoàn thành:
- Output score có cả `signal_score` và `total`.
- App vẫn chạy như trước.
- Không mất backward compatibility.
- Không có logic mới ngoài Phase 1.

Sau khi sửa, hãy báo cáo:
1. File đã sửa.
2. Function đã sửa.
3. Nơi đã thêm `signal_score`.
4. Nơi đã đổi cách đọc từ `total` sang fallback `signal_score -> total`.
5. Test nên chạy.
6. Có phần nào chưa sửa vì nằm ngoài phạm vi prompt này không.

Nhắc lại:
Chỉ chỉnh sửa đúng phần được yêu cầu. Tuyệt đối không sửa, refactor, đổi tên, format lại hoặc tối ưu các phần không liên quan.
