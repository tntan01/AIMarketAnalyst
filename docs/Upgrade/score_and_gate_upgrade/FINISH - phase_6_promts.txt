Bước 2 — Thêm hàm tính spread cost đơn giản:

```python
def calculate_spread_cost(spread_price: float | int | None) -> float:
    ...
```

Logic:
- Nếu `spread_price` là None hoặc không convert được sang float thì trả 0.0.
- Nếu `spread_price < 0` thì trả 0.0.
- Nếu hợp lệ thì trả float(spread_price).

Gợi ý:

```python
def calculate_spread_cost(spread_price):
    try:
        value = float(spread_price or 0.0)
    except (TypeError, ValueError):
        return 0.0
    return max(value, 0.0)
```

Bước 3 — Thêm hàm expected effective R:R:
Trong `core/risk_engine.py`, thêm:

```python
def calculate_expected_effective_rr(
    direction: str,
    entry: float,
    stop_loss: float,
    take_profit: float,
    spread_price: float = 0.0,
) -> float:
    ...
```

Logic bắt buộc:

```python
direction = str(direction).lower()

if direction == "buy":
    risk = abs(entry - stop_loss)
    reward = abs(take_profit - entry)
elif direction == "sell":
    risk = abs(stop_loss - entry)
    reward = abs(entry - take_profit)
else:
    return 0.0

spread_cost = calculate_spread_cost(spread_price)

effective_risk = risk + spread_cost
effective_reward = reward - spread_cost

if effective_risk <= 0:
    return 0.0

return max(effective_reward / effective_risk, 0.0)
```

Yêu cầu:
- Hàm không được raise exception khi input lỗi.
- Nếu entry/SL/TP None hoặc không convert được thì trả 0.0.
- Nếu direction không phải buy/sell thì trả 0.0.
- Nếu reward sau trừ spread âm thì trả 0.0.
- Kết quả nên round hợp lý, ví dụ 4 chữ số thập phân, nhưng không bắt buộc nếu test dùng `pytest.approx`.
- Không dùng thư viện ngoài.
- Không làm thay đổi các hàm risk cũ.
- Không đổi position sizing.
- Không đổi lot calculation.
